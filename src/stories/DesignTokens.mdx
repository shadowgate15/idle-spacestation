# Design Tokens

This page is a working baseline for upcoming theming work. It documents the Tailwind utility scales currently available in Storybook today, which at the moment come from Tailwind’s default v4 setup plus the existing `forms` and `typography` plugins loaded in `src/routes/layout.css`.

## How to use this page

- Treat these as the current foundation, not a finalized design system.
- Use the utility names shown here as the contract we can later remap to project-specific theme values.
- Extend this page as custom color roles, semantic tokens, and CSS variables are introduced.

## Colors

The project does not define custom brand colors yet, so this section highlights a few useful default Tailwind families to support early UI exploration and future token mapping.

<div className="tokens-section">
	{colorFamilies.map((family) => (
		<div className="token-card" key={family.name}>
			<div>
				<h3>{family.name}</h3>
				<p>Default Tailwind color scale available today for early theming exploration.</p>
			</div>
			<div className="color-grid">
				{family.chips.map((chip) => (
					<div className="color-chip" key={chip.label}>
						<div className={`color-swatch ${chip.className}`} />
						<div className="token-meta">
							<code>{chip.label}</code>
						</div>
					</div>
				))}
			</div>
		</div>
	))}
</div>

## Spacing

Spacing utilities are one of the most important scales to preserve during theming because they shape rhythm across layout, components, and dense application surfaces.

<div className="token-card">
	<div>
		<h3>Spacing scale</h3>

			Previewing the relative step sizes makes it easier to choose a consistent rhythm before tokens
			become semantic.
		</p>
	</div>
	<div className="spacing-stack">
		{spacingScale.map((step) => (
			<div className="spacing-row" key={step.label}>
				<div className="spacing-label">

					<code>{step.gap}</code>
				</div>

					<div className={`spacing-bar ${step.width}`} />
				</div>

		))}
	</div>
</div>
